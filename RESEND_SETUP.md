# הגדרת Resend לשליחת אימיילים

## הבעיה הנוכחית:

Resend מאפשר בשלב הבדיקה לשלוח **רק לכתובת האימייל שלך** (haimaimon5@gmail.com).

כדי לשלוח אימיילים לכתובות אחרות, אתה צריך **לאמת דומיין** ב-Resend.

## פתרון - אימות דומיין:

### שלב 1: הירשם ל-Resend ואימות דומיין
1. היכנס ל-Resend: https://resend.com/domains
2. לחץ על "Add Domain"
3. הזן את הדומיין שלך (למשל: `yourdomain.com`)
4. עקוב אחרי ההוראות להוספת DNS records

### שלב 2: עדכן את כתובת השולח

לאחר שהדומיין מאומת, עדכן את הקובץ `convex/emails.ts`:

בשורה 28, שנה מ:
```typescript
from: "Visa Morocco <onboarding@resend.dev>",
```

ל:
```typescript
from: "Visa Morocco <noreply@yourdomain.com>", // או כל כתובת אחרת מהדומיין שלך
```

### שלב 3: בדיקה

לאחר העדכון:
- האימיילים יישלחו לכל כתובת אימייל
- לא תהיה עוד שגיאת 403

## פתרון זמני (לבדיקה):

אם אתה רוצה לבדוק עכשיו, אתה יכול לשלוח אימייל לעצמך:
- כל האימיילים יישלחו ל-`haimaimon5@gmail.com`
- זה יעבוד לבדיקה אבל לא ללקוחות אמיתיים

**חשוב**: לפרודקשן, **חייב** לאמת דומיין!

## צעדים מהירים:

1. ✅ ה-API Key כבר מוגדר

לאחר ביצוע הצעדים, המערכת תוכל לשלוח אימיילים לכל כתובת!
